Traffic Flow Simulation

A traffic modeling project grounded in real-world street observations. Queue capacity, vehicle arrival rates, and signal timing were collected from actual intersections, then modeled in a C simulation engine, wrapped in a Python API, and presented through an interactive browser visualizer that renders streets, cars, lights, and congestion tick by tick.

How it was built
Real-world observation: visited local intersections to estimate street queue capacity (9-54 cars), count vehicle arrivals per 5-minute window, and note signal vs. stop-sign control for 12 street segments.
C systems design: modeled streets as structs with linked-list queues, implemented tick-based simulation with probabilistic car arrivals, signal phase alternation, stop-sign yield logic, and destination routing.
Edge-case testing: validated behavior across low/high simulation durations, full and empty queues, red-light blocking, stop-sign throughput, and destination-overflow scenarios.
Full-stack rebuild: wrapped the compiled C binary in a FastAPI backend with CORS, then built an interactive SVG visualizer showing roads, directional lanes, car blocks, signal states, and per-street detail inspection.
What the demo shows

The live demo runs the C engine through the Python API, then plays back 60 ticks of simulation (representing 5 minutes of real traffic). You can scrub through ticks, click any street to see its queue count, capacity, fullness percentage, and signal state. Peak congestion is highlighted automatically.

Architecture
Simulation engine
C binary accepts duration and seed, outputs JSON with 60 tick snapshots of the full street network state.
Backend API
FastAPI wraps the engine binary, exposes /run-simulation with query params, and serves JSON to the frontend.
Frontend visualizer
Static HTML/CSS/JS renders an SVG street scene with playback controls, summary stats, and a detail panel for inspecting individual streets.
Data source
12 observed street segments
Queue capacities from 9 to 54 cars, arrival rates from 3 to 42 cars per 5-minute window, mix of signal and stop-sign control.
